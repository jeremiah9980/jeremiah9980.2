Executive Summary

The sample MXToolbox-style scan returned 10 problems against soldbyshonna.com: 8 errors and 2 warnings, against 24 passing checks. That looks noisy, but the report identifies only one real critical risk.

This domain is intentionally forwarding-only. It does not host a mailbox or a production website. The real issue is that no SPF, DKIM, or DMARC records are published, which means attackers can forge email from a domain that no one is actively monitoring.

Findings & Recommended Actions

Critical

No anti-spoofing records on a parked-mail domain

A forwarding-only domain should explicitly declare that it sends no mail. Publish a null SPF record and reject DMARC so forged messages fail authentication. 

soldbyshonna.com TXT "v=spf1 -all"
_dmarc.soldbyshonna.com TXT "v=DMARC1; p=reject; rua=mailto:dmarc-reports@shonnaking.com; fo=1"
Medium

HTTP 403 on direct scanner hit should be verified in a browser

The audit treats this as likely scanner noise caused by GoDaddy forwarding behavior, but the redirect should still be tested manually from a clean browser and mobile device.

Low

SOA Expire value outside recommended range

This is cosmetic for this use case. Adjust at the DNS host during a routine cleanup window if desired.

Ignore

Missing MX / DKIM / BIMI findings are mostly scanner noise

Because this domain is intentionally not used for mailbox hosting, the important remediation is not to add mailboxes; it is to publish policy records that prevent abuse.
